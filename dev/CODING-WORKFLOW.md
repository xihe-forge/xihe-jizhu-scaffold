# Coding Workflow

## Principles

- Prefer small tasks with clear acceptance criteria.
- Keep shared code small and intentional.
- Add verification before declaring completion.
- Update planning state when scope changes.

## Defaults

- TypeScript strict mode
- Review before completion
- Verification required for feature changes
- Bug fixes must include root cause notes

## Done Definition

A change is complete when:

- the task acceptance criteria are met
- a build, test, or verification command was run
- the relevant state files are updated
